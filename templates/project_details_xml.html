Generating Database XML-Files

Problem:

All the assets we stored had to have a "scanned date" or else it was not considered valid. So for example if you updated the position manually in the program it would not get that date, since it would have to be physically scanned.

A problem occurs if it is a serial number/code that is inside the chassi of a machine (e.x battery units) we would not be able to physically scan those since we would have to power off the machine.


Solution:

I wrote a commandline script that we could use to enter serial numbers manually and generate the same XML-files the scanner would do, enabling us to import it to our database program as normal.


Technologies used:
